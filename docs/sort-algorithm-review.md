# ソートアルゴリズムレビュー

## 現在の方式

対話型の5分割クイックソートで、配列全体ではなく上位K件に関係する領域だけを再帰的に確定します。比較結果は次の意味です。

- `-2`: 左を強く好む
- `-1`: 左を好む
- `0`: 引き分け
- `1`: 右を好む
- `2`: 右を強く好む

## 今回確認・修正した点

- 0件と1件を明示的に処理し、1件では比較なしで順位1を設定
- pivotの返値を配列ではなく数値indexに統一
- 5領域の向きをテストで固定（右を好む回答は右の曲を上位側へ配置）
- Top-K境界をまたぐ領域だけを再帰し、境界外は比較を省略
- 同順位回答はpivotと同じ順位へまとめる
- 乱数をseeded randomへ変更し、同じ回答履歴から同じ比較順を再現可能にした
- UIイベントを比較ごとに追加せず、固定ハンドラーで処理
- 同じ曲ペアは左右の順序を正規化し、過去の回答を符号反転も含めて再利用

自動テストでは、0/1件、複数seedでのTop-K、全曲同順位、乱数再現性、重複なし乱数抽出を確認しています。

## 性質と限界

1. **回答が推移的ならTop-Kを正しく求められる**
   A>B、B>CならA>Cという一貫した回答を前提にしています。
2. **非推移な好みは一意な順位を持たない**
   A>B、B>C、C>Aのような回答では、どの比較ソートでも厳密な全順序を保証できません。
3. **5段階の強さが領域分割へ直接影響する**
   1/5を使うとpivotから遠い領域に入るため比較数は減りやすい一方、入力ミスの影響が大きくなります。
4. **同順位はpivotとの局所的な同順位**
   複数曲がそれぞれpivotと同順位なら同じrankになりますが、曲同士を直接比較してはいません。
5. **進捗表示は推定値**
   未比較領域のrank確定数を立方根で補正しており、残り比較回数を厳密には表しません。

- 実測した比較回数から進捗を推定する

これらは結果の意味や操作感が変わるため、実装前に仕様決定が必要です。
